# 语言与领域知识参考

处理特定语言/领域的评估时，参考本文件中的关键关注点。这些不是死记硬背的检查清单，而是帮助识别潜在问题的知识索引。

---

## 语言特有关注点

### Go

| 关注点 | 典型问题 | 评估要点 |
|--------|----------|----------|
| Goroutine 泄漏 | 无 context 取消、channel 未关闭 | 是否有明确的生命周期管理 |
| 错误处理 | err 被忽略、错误信息不携带上下文 | 是否 wrap error、是否有 sentinel error 设计 |
| Context 传播 | context 断链、超时未传递 | 跨层调用是否正确传递 context |
| Interface 设计 | 接口过大、消费者未定义接口 | 是否遵循"接口属于消费者"原则 |
| 并发安全 | map 并发读写、race condition | 是否需要 sync.Mutex/RWMutex/atomic |
| 依赖注入 | 全局变量耦合、测试困难 | 构造函数注入 vs 全局单例 |

### Python

| 关注点 | 典型问题 | 评估要点 |
|--------|----------|----------|
| GIL | CPU 密集任务假并发 | 是否应使用 multiprocessing 或 C 扩展 |
| 类型提示 | 运行时类型不匹配、IDE 无法推断 | 关键接口是否有 type hints |
| 依赖管理 | 版本冲突、依赖地狱 | 是否锁定版本、是否隔离环境 |
| Async 陷阱 | 阻塞调用混入 async、忘记 await | 是否全链路 async、是否有阻塞检测 |
| 内存 | 大对象未释放、循环引用 | 是否需要 weakref、generator |
| 包结构 | 循环导入、层级混乱 | 是否有清晰的分层和依赖方向 |

### C++

| 关注点 | 典型问题 | 评估要点 |
|--------|----------|----------|
| RAII | 资源泄漏、裸指针管理 | 是否用智能指针、是否有确定性析构 |
| Move 语义 | 不必要的拷贝、move 后使用 | 是否正确实现移动构造/赋值 |
| ABI 兼容 | 跨编译单元的类布局变化 | 是否需要 pimpl、是否暴露了实现细节 |
| 模板 | 编译时间爆炸、错误信息不可读 | 是否过度使用、能否用 concept 约束 |
| 内存模型 | 多线程下的 data race | 是否需要 atomic、memory_order 是否正确 |
| 异常安全 | 构造函数抛异常后的状态 | 基本保证 / 强保证 / 不抛出 |

### Lua

| 关注点 | 典型问题 | 评估要点 |
|--------|----------|----------|
| GC 压力 | 频繁创建临时表、字符串拼接 | 是否对象池化、是否预分配 |
| 元表设计 | 继承链过深、__index 查找开销 | 是否扁平化、是否缓存方法 |
| C 交互 | 栈操作错误、类型不匹配 | 栈平衡是否检查、userdata 生命周期 |
| 热更新 | 状态丢失、引用残留 | 是否有 reload-safe 设计、upvalue 处理 |
| 全局污染 | 意外全局变量、命名冲突 | 是否 local 化、是否有全局检测 |

## 领域关注点

### 后端服务

| 维度 | 关注点 |
|------|--------|
| 一致性 | 分布式事务、最终一致 vs 强一致、补偿机制 |
| 容错 | 熔断、降级、重试策略（指数退避、幂等性） |
| 可观测性 | 链路追踪、指标采集、日志分级、告警策略 |
| 限流降级 | 令牌桶/漏桶、优先级队列、过载保护 |
| 数据存储 | 分库分表、缓存策略、读写分离 |

### 前端

| 维度 | 关注点 |
|------|--------|
| 包体积 | Tree shaking、动态 import、依赖分析 |
| 渲染性能 | 虚拟列表、防抖节流、重绘回流 |
| SSR/CSR | 首屏时间、SEO 需求、hydration 成本 |
| 状态管理 | 全局 vs 局部、持久化、时间旅行调试 |

### 客户端/游戏

| 维度 | 关注点 |
|------|--------|
| 帧率 | 主线程阻塞、GC 停顿、Draw Call 合批 |
| 内存 | 纹理压缩、对象池、分帧加载 |
| 热更新 | 代码热更 vs 资源热更、回滚机制 |
| 对象池 | 回收策略、预热数量、容量上限 |

## 跨语言复合场景

### 常见组合

| 组合 | 场景 | 核心关注 |
|------|------|----------|
| C++ + Lua | 游戏引擎 + 脚本 | 绑定层开销、GC 对帧率的影响 |
| C# + Lua (Unity) | xLua 桥接 | GC Alloc、IL2CPP 限制 |
| Go + C++ | 后端高性能模块 | cgo 开销、gRPC 协议设计 |
| C++ + Python | ML 推理 + 系统 | pybind11 边界、GIL 阻塞 |
| Go + Python | 微服务混合 | 序列化协议、错误码映射 |

### 🔴 跨语言边界必查清单

处理任何跨语言复合场景时，以下 5 项必须逐一检查并在输出中 🔴 高亮：

| # | 检查项 | 关注什么 |
|---|--------|----------|
| 1 | 绑定层性能 | 调用频率 × 单次开销，是否在热路径上 |
| 2 | 数据拷贝 | 跨边界是否零拷贝，序列化/反序列化开销 |
| 3 | 生命周期 | 谁分配谁释放，跨语言 GC 能否正确回收 |
| 4 | 错误传播 | 异常/error 如何跨语言传递，是否丢失信息 |
| 5 | 线程模型 | 跨语言调用是否阻塞主线程，回调线程是否安全 |
